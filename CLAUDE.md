# Правила работы с проектом ТОНУС

## Архитектура

- Весь сайт — один файл `index.html`. CSS внутри `<style>`, JS внутри `<script>`.
- Никаких внешних библиотек (jQuery, Bootstrap, Swiper и т.п.). Только ванильный JS и чистый CSS.
- Google Fonts подключаются через `<link>` — это не библиотека, это шрифты.
- SVG-иконки — inline прямо в HTML.

## Дизайн

- Светлая, чистая медицинская эстетика. Дизайн определён в `brief.md` и `thoughts/specs/idea.md`.
- Цветовая палитра: белый фон (`#FFFFFF`), мягкий голубоватый (`#F4F8FC`), акцентный синий (`#2B7FBF`), тёплый бирюзовый (`#4EADA0`), тёмный текст (`#1A2B3C`).
- Шрифты: Manrope (заголовки) + Inter (текст). Оба с поддержкой кириллицы.
- Без тёмных фонов, резких цветов и перегруженных блоков.

## Адаптивность

- Mobile-first: всё должно хорошо выглядеть на экранах от 320px.
- Touch-зоны: минимум 44×44px для кнопок и ссылок.
- Все телефонные ссылки через `<a href="tel:...">` — обязательно.
- Sticky bottom bar появляется на мобильных после скролла hero.

## Стиль текста

- Простой, живой, дружеский язык. Говорим с клиентом (гостем) как с человеком.
- Без канцелярита: не «осуществляется помощь», а «помогаем».
- Без лишних слов: не «в рамках данного подхода», а просто «здесь».
- Заголовки — короткие и конкретные. Текст в карточках — 2–3 предложения максимум.

## Лексика — обязательные замены

| Нельзя | Правильно |
|--------|-----------|
| пациент | клиент, гость |
| приём | сеанс |
| лечение | оздоровление, восстановление, коррекция |
| лечим / лечится | помогаем / восстанавливается |
| врач (о специалисте студии) | специалист, специалист по реабилитации |
| доктор (о специалисте студии) | специалист |

**Специалист:** Мария Федина — специалист по реабилитации и восстановлению.  
**Важно:** работа начинается с первого сеанса, но результат не гарантирован сразу — не обещать «вылечим с первого раза».  
_(«врача» как внешней профессии — направление от врача, МРТ — оставлять как есть)_

## Перед созданием файла

- Сначала рассказать, что планируется создать и зачем.
- Дождаться подтверждения — и только потом писать файл.
- Это правило касается любого нового файла: HTML, CSS, JS, изображения, конфиги.

## Главные документы проекта

- `brief.md` — главное ТЗ. Перечитывать перед любой правкой дизайна или контента.
- `thoughts/specs/idea.md` — исходная идея и структура блоков.
- `research.md` — исследование: конкуренты, тренды, лучшие практики.
